python寫檔的thread-safe問題

發表於 2021-06-06 分類於 ceiere_hide

假設檔案就一個, 根據寫檔的來源考慮以下兩種情況:

  • 多個thread
  • 多個process

多個thread

這個好解, 因為開thread代表你有一個process, 可以對生出來的thread做控制,
我可以在寫檔案的區塊上鎖就好了

或是直接使用python內建的logging, 本身就是thread-safe的

多個process

這個比較棘手一點, 又可以分成兩種情況:

  • 原本有一個process(主程式), 透過multiprocessing開process
  • 多開程式

第一種情況起碼還可以對process做掌控, 透過multiprocessing的Queue,
概念是process往Queue去丟訊息, 此外需要一個consumer去從Queue拿訊息
跟Message Queue概念很像

第二種就麻煩了(偏偏我遇到的是這一種啊…)
因為是不同程式, process是OS開的, 這個就很難去對process做控制
解法就是透過Server(目前還沒找到其他解法不屬於這個的)

你開一個Server, 不同process寫log會呼叫server一個API, Server端做寫入的動作
或是你也可以用官方文件說的, 使用SocketHandler, 但也是要建SocketServer意思一樣
或是你用Message queue, 那還是一樣有Server

延伸閱讀

那Flask是如何做到logging的, 如果Gunicorn開4個worker(4 process)
其實也挺直覺, 因為Gunicorn本身就有一個master process, 全交由他寫入即可

物件生命週期

new, init, metaclass, 這三個可以在類創建,實例創建和實例初始化的時候做一些小手腳.

參數傳遞

如何知道變數的記憶體位址? id(var)
如何比較地址? 用is比較地址, 用==比較值

python所有東西都是object, 根據能否更改可分為mutable, immutable

mutable:

  • string
  • number
  • tuple,

immutable:

  • list
  • dict
  • etc…

這個現象會反應在參數傳遞上

因為number是immutable, 傳入function後,
若更改值, 則會分配一個新的記憶體位置, 所以val和var不互相影響

因為list是mutable, 傳入後就算對他更動, 還是參考相同的記憶體位址
因為foo不用回傳, val的值亦變成[1, 2, 3, 4]

結論是:

  • python是call by assignment
  • 傳immutable object進去function, 行為會像call by value
  • 傳mutable object進去function, 行為會像call by reference

@staticmethod和@classmethod

static_foo: staticmethod代表類別方法, 只會存一份在memory, 所有object共享. 不需產生物件即可使用

foo: 物件方法每個object各自維護, self是代表把目前的物件傳進去
class_foo: classmethod類似staticmethod, 不用new可用, 因為class定義本身也是物件, 第一個參數代表的就是該類別的物件(藍圖本身也放在記憶體內一個區段)

class variable

正常來說, 物件的變數我們會放到init內, 那如果是放在class內, function外呢?
代表該變數是class variable, 該變數屬於class, 所有object共享
這種做法可以用來打造singleton, 或是當成所有物件的狀態紀錄(e.g. counter)

underline

下底線如果是兩個, 代表是python內部的變數, 一般而言不太會去修改
e.g. init

如果是一個, 約定俗成代表該變數是private(因為python沒有private保留字)
e.g. _abc

開頭兩個下底線, interpretor會改名, 如此則不能直接存取(其實更符合private特性)
e.g. atr, 會被改成_classnameatr,

其中比較重要的幾個欄位有:

  • _scroll_id: 後續的分頁查詢須帶這個id
  • _shards: 該次query使用了多少shard

後記:

其實我花最多時間的部分在於查詢到底search跟scroll API要給一個scroll參數, 這是代表啥意思?
他其實是代表該次query之後這個view會維持多久, 以上面例子來說, 2m等於會維持2分鐘.

再白話一點就是這兩分鐘之內, 如果你的資料有被異動, 你不會拿到新的資料,
這樣做也是挺合理的,
我可能每次翻頁後要做一些耗時的IO操作, 可能全部翻完頁要花5m, 這樣我就可以設定scroll=1m-5m,
確保es給你的資料是程式開始執行瞬間的資料,
上面的scroll給1m-5m資料範圍是因為實際上要根據你總共翻幾頁, 要省事一點就直接設5m,
不然其實每次翻頁(call scroll)時帶的scroll參數其實都會再刷新view維持的時間.
