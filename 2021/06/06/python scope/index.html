python Scope

發表於 2021-06-06 分類於 ceiere_hide

直接看例子

印出:
4
456
123
local內可存取global的值

印出:
456, 因為無法更改global的值

印出:
123, 使用global可以更改值

印出7, a此時是local var

只是把兩行對調就噴錯訊, 為何第一個print不會印出global的a?
原因是在function在一行一行執行code前會先掃var,
知道有對a assign值後認定a是local var,
然而在第一行看到你要對未定義的a做print, 就噴錯了

印出not defined, 因為a是local

印出123, 使用global後, a被提升到global, 就算沒在外部去宣告

印出1,2, nonlocal可以存取非global, 也非local的, 那就是存取外層local var

印出[4], [4,5], list是mutable, 所以只維護一份

接下來三題要掌握一個原則:
value是根據function建立時期綁定, 還是呼叫時期

印出changed, original 我的天…

第一個印changed, 原因是呼叫時的值, 已經被綁到changed
第二個印original的原因是,
function可以簡單想成兩步驟: 建立, 被呼叫
而function parameter default value在建立function時就會綁定值
程式從上而下執行, 所以s綁定了original

印出12, 12
原因是a值在呼叫時綁定, 等到呼叫時a是2
所以印出12, 12

一樣印出12, 12
