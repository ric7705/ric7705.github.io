static method, class method

其實跟JAVA靜態方法類似, 定義在類裡面的method是屬於class, 而非obj
所以實體化後的物件不會綁這個method(因為method是綁類別)

diff:

  • @classmethod: 第一個argument是類別
    • 使用時機: 如果需要access類別屬性或類別本身時
  • @staticmethod: 沒有第一個argument
    • 使用時機: 完全不需要access類別屬性或類別本身時

使用類別或靜態方法的優點:

  • 效率較高, 不必為每個物件都bound這個方法(在python中function也是obj)
  • 可讀性提昇, 因為看到static method就知道這func做的事跟物件狀態無關

嘗試1: @classmethod第一個引述命名有限制嗎? 跟self的區別?
無限制, 但是按照慣例會取cls(class縮寫), this class(多個物件還是指相同位址)
和self不太一樣, self比較像 this object(可能有多個物件, self指不同位址)

嘗試2: classmethod或staticmethod我也可以透過物件的方式取用嗎?
A: 可以的, 要new不new都可以

嘗試3: 看一下繼承規則, 父子之間overide的行為是怎樣?
當遇到繼承時, 行為跟一般function一樣, 子類會蓋掉父類

使用時機

  • @classmethod: 需要access類別屬性或類別本身時
    • multi constructor
    • factory method
  • @staticmethod: 完全不需要access類別屬性或類別本身時
    • 寫一個公用utility Class
    • 只有該類別會用到, 但是又跟物件狀態無關的function
    • 如果有參數,物件產生的, 也可以用嗎?

class attribute使用時機

  • 當常數用, ex: pi=3.14 (但其實沒有final的效果)
  • 當預設值用: 物件如果沒給值, 則預設使用Class的
  • 物件之間share相同scope: 例如share相同list, dict等
    • 這用法很強大, 但是要注意整個程式內不可以對這變數使用到這種形式’obj.attribute=value’,否則此時obj開始各自維護變數, 邏輯變得很雜
